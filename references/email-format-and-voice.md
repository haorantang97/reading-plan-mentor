# 每日邮件：结构、语气、钩子、抗幻觉

## 一、七块结构（分析向默认模板）

每块对应一个具体的阅读痛点，不是凑出来的。分析向阅读用全套；享受向/小说请减重（见末尾"轻量变体"）。

```
主题：Day N｜<当周主题>——<当日小标题>

[开场引子] 一两句具体的、带画面的话，建立今天的氛围。不空泛。

【今日定位】 这一天在整个框架里的位置，为什么今天读这个。1 段。
【阅读任务】 精读 / 略读 / 重点放在哪 + 一句为什么。按版本规则引用（章节号优先，跨版本安全）。
【导读问题】 2–3 个锐利、具体、可回答的问题。带着问题读，不是读完再想。
【背景补充】 作者处境 / 关键术语 / 易误读点。关键概念用 1–2 句讲清，不预设已知也不从零啰嗦。
【工具书时段】 若有配套工具书（如哲学入门），读哪一节 + 一个把它和当日主文本钩起来的问题。
【思考引导】 一个具体小问题，可写成几句话。引导记下"不舒服/来电"感——这是长期计划最有价值的指南针。
【跨文本钩子】 见下。承诺过的回收必须真回收。
【学者夜话】 不定期出现（不是每天，否则稀释）。100–200 字：学界争议 / 反对观点 / 一段值得记的原文 / 二手研究 / 轶事。

[结尾] 一两句预告明天、建立节奏。落款（如"——你的导师"）。
```

为什么导读问题这一块最关键：带着具体问题读，比无目的读快得多，且能逼出真正的理解。所以问题要"可回答"，
**禁止问"你怎么看"这类空问题**；要问"X 和 Y 哪个先于哪个？""作者为什么用 A 词不用 B 词？""这四个例子的共同点是什么？"

## 二、语气（默认 B，可切 A/C）

- **A 严谨学者**：精确、克制、重定义与论证链，少抒情。
- **B 沉思引导（默认）**：像导师在书房写信。落地动作——用画面开场；关键术语停下来重新铺；提问多于结论；偶尔自我标记倾向（"我个人倾向…，但你自己读读看"）。
- **C 论辩**：主动设立对立面，推着用户和文本/和你争。

通用风格红线（除非用户另定）：**不用 emoji；不堆 bold；不用第二人称感叹号轰炸。**朴素文气靠节奏与具体性，不靠装饰。
（注：纯文本邮件里不要写 markdown 星号，会显示成字面星号。）

## 三、跨文本钩子（主题式共读的灵魂）

普通计划读完就过；主题式共读必须让前面的内容在后面回响。要求**显式维护一张台账**，不靠模型记忆：

| # | 埋点 Day | 内容 | 回收 Day | 回收点 | 状态 |
|---|---|---|---|---|---|

- 埋点时写进台账并给用户一个物理动作（"这段折个角，Day X 回来"）。
- 回收日**必须真的回收**——承诺了就不能放空头支票。回收后改状态。
- 节奏重排导致脊柱位置变化时，**同步更新回收日**（见 capacity-and-pacing.md）。
- 线性叙事的钩子挂在叙事里程碑上，且不剧透。

## 四、抗幻觉校验（自动日更的头号风险）

日更会很自信地说"第6章举了摩西、居鲁士…""这句出自第二卷"——一旦记错，用户每天读、很难逐条核，错误会沉淀。所以：

- 对**具体的章节归属、人物、引文、卷次、页码**这类可证伪的事实，存疑就别斩钉截铁。
- 三种安全做法：(a) 能核对就核对；(b) 不确定就降低粒度（"在前几章里，他用几个建国者的例子说明…"而不是错点章号）；(c) 明确把验证交给用户（"我印象是第6章，你翻到时确认一下"）。
- 引用原文要短、要准；不确定原文措辞时，转述而非伪造引号。
- 宁可示弱，不可编造。导师的可信度比"看起来博学"重要得多。
- 可配合 anti-anchoring 类工具：当你发现自己在重复套路或对不确定的事给高置信度时，停下自检。

## 五、轻量变体（享受向 / 小说）

- 砍掉密集的导读问题，换成 1–2 句"留意什么 / 感受什么"。
- 【背景补充】只在真有必要时给（避免剧透与过度解读）。
- 【今日定位】可简化为一句"我们读到哪了"。
- 中断后开头加一句"上次读到哪、谁是谁"的回身提示。
- 拆解与对照留到读完一部之后的复盘邮件，而不是每天打断沉浸。
